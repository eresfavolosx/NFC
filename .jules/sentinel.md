## 2024-05-24 - Widespread Stored XSS via innerHTML Interpolation
**Vulnerability:** The application was vulnerable to severe Stored Cross-Site Scripting (XSS). User inputs (such as link titles, URLs, and tag labels) were directly interpolated into raw HTML strings and then injected into the DOM using `innerHTML` across multiple views (e.g., `links.js`, `tags.js`, `dashboard.js`, `writer.js`).
**Learning:** The vulnerability existed because the project uses vanilla JavaScript with template literals for rendering views without a framework-level sanitization mechanism (like React or Vue provides by default). Developers prioritized developer experience by building HTML strings, inadvertently bypassing standard DOM text node protections.
**Prevention:** Always sanitize untrusted user data before interpolating it into HTML templates. Use a utility function like `escapeHTML` to escape characters (`&`, `<`, `>`, `"`, `'`) or prefer assigning user data to `textContent` directly when possible.
