## 2026-04-08 - Avoid N+1 Map Pre-fetching Micro-optimizations
**Learning:** Pre-fetching Maps and replacing simple property getter calls inside loops with closures or pre-fetched map references is considered a 'useless micro-optimization' and an architectural anti-pattern in JS, as property access is extremely fast. Additionally, attempts to provide default fallbacks using arrays (e.g., `[]`) inside those loops degrades memory by allocating unused arrays.
**Action:** Do not attempt to optimize O(N) property getters that rely on internal lazy-caching Maps. If a fallback array is absolutely necessary, subclass the `Map` to return a shared `Object.freeze([])` instance to avoid memory bloat.
## 2024-05-01 - Avoid NodeList caching and forEach loop micro-optimizations
**Learning:** Attempting to optimize UI filtering by caching `document.querySelectorAll()` results in a module-level variable introduces severe state bugs (stale cache risk) when items are added/removed. Additionally, replacing `.forEach()` with a standard `for` loop on small NodeLists is a rejected micro-optimization that sacrifices readability for zero measurable gain.
**Action:** Do not cache DOM queries unless explicitly handling lifecycle synchronization. Never replace `.forEach()` loops with `for` loops merely for theoretical performance, as it violates the guidelines against micro-optimizations. Focus on algorithmic improvements (like reducing O(N) evaluations in `.filter()`) instead.
